Pattern:
    Two pointers with state tracking

Also solvable using:
    Prefix arrays (DP)
    Monotonic stack

But optimal:
    Two pointers + running maximums

This is NOT a local problem.You cannot look at:
    height[i-1], height[i], height[i+1]
Instead:
    Water depends on GLOBAL boundaries

At any index:
    water[i] = min(max_left, max_right) - height[i]

We track:
    i → left pointer  
    j → right pointer  
    leftMax → highest wall seen from left  
    rightMax → highest wall seen from right


Decision Rule (CRITICAL)-
    If height[i] < height[j]:
        → process left
    Else:
        → process right



Trigger conditions:
1. Water / area / accumulation problems  
2. Need left + right boundary  
3. Global constraint, not local  
4. Want O(n) without extra space
